이 문서는 MSW 프로젝트를 처음 만들 때 기본으로 세팅된
"플레이어가 공격하면 몬스터가 데미지를 입고 반응하는" 기능을
다른 프로젝트에 이식하기 위해 이해하는 용도로 작성했습니다.

----------------------------------------------------------------
  1. 전체 흐름 요약
----------------------------------------------------------------

[플레이어가 공격키 입력]
        ↓
  PlayerActionEvent 발생 (ActionName = "Attack")
        ↓
  PlayerAttack.mlua 가 이벤트를 받아 AttackNormal() 실행
        ↓
  BoxShape(공격 범위)를 계산하고 AttackFast() 호출
        ↓
  Monster 충돌 감지 → HitEvent 발생
        ↓
  Monster.mlua 가 HitEvent를 받아 HP 감소 처리
        ↓
  HP가 0 이하이면 Dead() → 사망 애니메이션 → 비활성화
        ↓
  RespawnOn = true이면 일정 시간 후 Respawn()

[몬스터가 플레이어를 공격]
  MonsterAttack.mlua 가 반복 타이머로 AttackNear() 실행
        ↓
  몬스터 스프라이트 크기 = 공격 범위로 사용
        ↓
  Player 충돌 감지 → HitEvent 발생
        ↓
  PlayerHit.mlua 가 무적 시간(ImmuneCooldown) 체크 후 처리

----------------------------------------------------------------
  2. 파일별 역할 상세 설명
----------------------------------------------------------------

[ PlayerAttack.mlua ]
파일 경로: RootDesk/MyDesk/PlayerAttack.mlua
상속: AttackComponent (MSW 내장)
실행 위치: @ExecSpace("ServerOnly") - 서버에서만 실행

  핵심 기능:
  - 플레이어의 공격 처리 담당
  - HandlePlayerActionEvent: 플레이어가 "Attack" 액션을 취하면 호출
  - AttackNormal(): 실제 공격 범위(BoxShape)를 만들어 판정 실행
  - CalcDamage(): 공격 데미지 계산 (기본 50)
  - CalcCritical(): 크리티컬 여부 계산 (기본 30% 확률)
  - GetCriticalDamageRate(): 크리티컬 배율 (기본 2배)

  중요 코드:
    self.Shape = BoxShape(Vector2.zero, Vector2.one, 0)
      → 1x1 크기의 사각형 공격 판정 박스 생성

    local attackOffset = Vector2(
        worldPosition.x + 0.5 * playerController.LookDirectionX,
        worldPosition.y + 0.5
    )
      → 플레이어가 바라보는 방향(LookDirectionX)으로 앞쪽에 공격 범위 배치
      → x: 바라보는 방향으로 0.5 유닛 앞
      → y: 플레이어 중심에서 위로 0.5 유닛 (가슴 높이)

    self:AttackFast(self.Shape, nil, CollisionGroups.Monster)
      → Monster 충돌 그룹에 속하는 대상만 맞춤

  다른 프로젝트로 이식 시 주의:
  - 공격 데미지(50), 크리티컬 확률(0.3), 크리티컬 배율(2)은 필요에 따라 수정
  - attackOffset: 플레이어 크기나 게임 설계에 맞게 조정 필요
  - CollisionGroups.Monster: 맞는 대상의 충돌 그룹 이름 확인 필요

[ Monster.mlua ]
파일 경로: RootDesk/MyDesk/Monster.mlua
상속: Component (MSW 기본)
실행 위치: 혼합 (서버/클라이언트)

  핵심 기능:
  - 몬스터의 HP 관리, 사망, 리스폰 처리
  - @Sync 속성들은 서버↔클라이언트 자동 동기화

  주요 속성:
    MaxHp = 100         → 최대 체력
    Hp = 0              → 현재 체력 (OnBeginPlay에서 MaxHp로 초기화)
    RespawnOn = false   → 리스폰 여부 (false면 사망 후 영구 제거)
    IsDead = false      → 사망 상태 여부
    RespawnDelay = 5    → 리스폰까지 대기 시간 (초)
    DestroyDelay = 0.6  → 사망 후 숨기기까지 대기 시간 (초, 사망 모션용)

  HandleHitEvent: 공격을 맞았을 때 호출
    → TotalDamage만큼 Hp 차감
    → Hp가 0 이하가 되면 Dead() 호출
    → RespawnOn이 true면 RespawnDelay 후 Respawn() 예약

  Dead():
    → IsDead = true 설정
    → StateComponent로 "DEAD" 상태 변경 (사망 애니메이션 재생)
    → DestroyDelay 후 Entity를 비활성화/숨김
    → RespawnOn=false면 Entity 완전 삭제(Destroy)

  Respawn():
    → IsDead = false 설정
    → Entity 다시 보이기/활성화
    → Hp = MaxHp (체력 완전 회복)
    → StateComponent로 "IDLE" 상태 변경

  다른 프로젝트로 이식 시 주의:
  - RespawnOn을 true로 설정하면 몬스터가 자동 리스폰 됨
  - StateComponent가 없어도 동작하지만, 사망 애니메이션은 안 나옴
  - @Sync 속성들은 멀티플레이 환경에서 자동으로 모든 플레이어에게 동기화됨
  - Dead()와 Respawn()은 @ExecSpace("ServerOnly") → 서버에서만 실행

[ MonsterAttack.mlua ]
파일 경로: RootDesk/MyDesk/MonsterAttack.mlua
상속: AttackComponent (MSW 내장)
실행 위치: @ExecSpace("ServerOnly") - 서버에서만 실행

  핵심 기능:
  - 몬스터가 플레이어를 공격하는 처리
  - 일정 시간 간격(AttackInterval)으로 반복 공격 판정

  주요 속성:
    AttackInterval = 0.03  → 공격 판정 반복 주기 (초, 기본 0.03초 = 매우 빠름)
    Shape                  → 공격 판정 박스 (몬스터 스프라이트 크기 기반)
    SpriteSize             → 스프라이트의 실제 크기
    PositionOffset         → 스프라이트 피벗 기반 위치 보정값

  OnBeginPlay() 실행 흐름:
    1. BoxShape 초기화
    2. SpriteRUID로 애니메이션 클립 비동기 로드
    3. 첫 번째 프레임의 스프라이트 크기 측정
       → 픽셀 크기 ÷ PixelPerUnit = 월드 유닛 크기
    4. 피벗 위치 기반으로 PositionOffset 계산
    5. SetTimerRepeat로 AttackInterval마다 AttackNear() 반복 호출

  AttackNear():
    → 몬스터의 현재 위치, 회전, 스케일을 고려해 공격 박스 위치 계산
    → self:AttackFast(self.Shape, nil, CollisionGroups.Player)
    → Player 충돌 그룹 대상으로 공격 판정

  IsAttackTarget():
    → 방어자(defender)에 PlayerComponent가 있는지 확인
    → 없으면 무조건 false 반환 (플레이어만 맞음)

  다른 프로젝트로 이식 시 주의:
  - AttackInterval이 0.03초로 매우 빠름 → 필요에 맞게 조정 (예: 1.0 = 1초마다)
  - 공격 범위를 스프라이트 크기 기반으로 자동 계산하는 구조
  - CollisionGroups.Player: 플레이어의 충돌 그룹 이름 확인 필요
  - IsDead 체크로 사망한 몬스터는 공격 안 함

[ PlayerHit.mlua ]
파일 경로: RootDesk/MyDesk/PlayerHit.mlua
상속: HitComponent (MSW 내장)

  핵심 기능:
  - 플레이어가 피해를 받을 때 처리 (피격 판정)
  - 무적 시간(ImmuneCooldown) 구현으로 연속 피격 방지

  주요 속성:
    ImmuneCooldown = 1   → 피격 후 다음 피격까지 최소 간격 (초)
    LastHitTime = 0      → 마지막 피격 시각 (게임 시작 후 경과 초)

  IsHitTarget():
    → 현재 시간 - 마지막 피격 시간 > ImmuneCooldown 이면 피격 허용
    → 피격 허용 시 LastHitTime 갱신 후 true 반환
    → 무적 시간 중이면 false 반환 (피해 무시)

  다른 프로젝트로 이식 시 주의:
  - ImmuneCooldown = 1 (1초 무적)을 게임에 맞게 조정
  - HitComponent를 상속하므로 HitEvent 자동 연동
  - 이 컴포넌트가 플레이어 Entity에 붙어 있어야 함

----------------------------------------------------------------
  3. 컴포넌트 관계도 (Entity에 붙는 방식)
----------------------------------------------------------------

  플레이어 Entity
  ├── PlayerControllerComponent  (MSW 기본 - 키 입력 처리)
  ├── PlayerAttack               (PlayerAttack.mlua - 공격 판정 발생)
  └── PlayerHit                  (PlayerHit.mlua - 피격 판정 처리)

  몬스터 Entity
  ├── Monster                    (Monster.mlua - HP/사망/리스폰 관리)
  ├── MonsterAttack              (MonsterAttack.mlua - 플레이어 공격)
  └── StateComponent             (MSW 기본 - 애니메이션 상태 관리)

----------------------------------------------------------------
  4. 핵심 MSW 내장 컴포넌트/이벤트 설명
----------------------------------------------------------------

  AttackComponent (내장):
  - AttackFast(shape, attackInfo, collisionGroup): 공격 판정 즉시 실행
  - CalcDamage(): 오버라이드로 데미지 계산
  - CalcCritical(): 오버라이드로 크리티컬 여부 결정
  - GetCriticalDamageRate(): 오버라이드로 크리티컬 배율 결정

  HitComponent (내장):
  - IsHitTarget(): 오버라이드로 피격 허용 여부 결정

  HitEvent:
  - TotalDamage: 최종 데미지 (크리티컬 배율 적용 포함)
  - AttackerEntity: 공격한 Entity
  - IsCritical: 크리티컬 여부
  - AttackCenter: 공격 중심점

  PlayerActionEvent:
  - ActionName: 액션 이름 ("Attack", "Jump" 등)
  - PlayerEntity: 액션을 취한 플레이어

  BoxShape(center, size, angle):
  - center: 박스 중심 위치 (Vector2)
  - size: 박스 크기 (Vector2, 월드 유닛)
  - angle: 회전 각도 (도)

----------------------------------------------------------------
  5. 다른 프로젝트에 이식하는 순서
----------------------------------------------------------------

  Step 1. 4개 .mlua 파일 복사
    → PlayerAttack.mlua
    → PlayerHit.mlua
    → Monster.mlua
    → MonsterAttack.mlua
    → 모두 RootDesk/MyDesk/ 경로에 배치

  Step 2. Maker에서 Refresh 실행
    → .codeblock 파일이 자동 생성됨
    → 스크립트가 MSW에 등록됨

  Step 3. 플레이어 Entity에 컴포넌트 추가
    → PlayerAttack 컴포넌트 추가
    → PlayerHit 컴포넌트 추가

  Step 4. 몬스터 .model 파일 설정 확인
    → Monster 컴포넌트 포함 여부 확인
    → MonsterAttack 컴포넌트 포함 여부 확인
    → StateComponent 포함 여부 확인 (사망 애니메이션용)
    → CollisionGroup이 Monster로 설정되어 있는지 확인

  Step 5. 수치 조정
    → Monster.mlua: MaxHp, RespawnOn, RespawnDelay, DestroyDelay
    → PlayerAttack.mlua: CalcDamage(50), 크리티컬 확률(0.3), 배율(2)
    → MonsterAttack.mlua: AttackInterval
    → PlayerHit.mlua: ImmuneCooldown

  Step 6. 테스트
    → 플레이어가 공격 시 몬스터 HP 감소 확인
    → 몬스터 HP 0 이하 시 사망 처리 확인
    → 몬스터 공격이 플레이어에게 닿는지 확인
    → 무적 시간 정상 동작 확인

----------------------------------------------------------------
  6. 자주 하는 실수 & 주의사항
----------------------------------------------------------------

  주의 1. CollisionGroup 이름 확인 필수
    - CollisionGroups.Monster, CollisionGroups.Player 이름이
      실제 프로젝트의 충돌 그룹 설정과 일치해야 함
    - 맞지 않으면 공격이 아무것도 맞추지 못함 (오류 없이 조용히 실패)

  주의 2. ServerOnly 실행 공간 유지
    - AttackFast(), Dead(), Respawn() 등은 서버에서만 실행
    - 클라이언트에서 호출하면 동작 안 함

  주의 3. MonsterAttack의 AttackInterval
    - 기본값 0.03초 = 초당 33번 판정 (매우 공격적)
    - 대부분의 게임에서는 0.5~2.0 정도로 조정 권장

  주의 4. TileMapMode와 Body 컴포넌트 일치 필수
    - 맵이 MapleTile(0): RigidbodyComponent
    - 맵이 RectTile(1): KinematicbodyComponent
    - 맵이 SideViewRectTile(2): SideviewbodyComponent
    - 불일치 시 Entity가 움직이지 않음 (오류 없이 조용히 실패)

  주의 5. StateComponent 없으면 사망 애니메이션 없음
    - Monster.mlua의 Dead()에서 ChangeState("DEAD")를 호출
    - 몬스터 .model에 StateComponent가 없으면 이 부분은 건너뜀
    - 사망 애니메이션을 원하면 StateComponent + ActionSheet 설정 필요
