# MSW 스킬 리소스 라이브러리 인덱스

MSW 리소스 라이브러리에 있는 **스킬 팩 전체**를 로컬에 덤프해둔 것. 스킬 리소스를 찾을 때
검색 운에 기대지 말고 **여기를 grep**한다.

> ## ⛔ 먼저 알 것: 여기에 "구버전 아트"는 없다 (실측 2026-07-17)
>
> 이 라이브러리는 **현대 메이플 클라이언트 덤프**다. WZ 스킬 ID는 버전이 바뀌어도 그대로라서
> **팩 ID가 구버전 경로여도 안의 그림은 현대에 새로 그린 것**이다.
>
> - `skill/111.img/skill/1111012`(크루세이더 돌진) → ID는 구버전, 아트는 현대 리메이크(주황 화살촉)
> - `skill/800033.img/skill/80003316`(파워 스트라이크) → 800xxx도 레거시 아카이브 **아님**, 현대 아트
>
> **사운드는 옛날 것이 그대로 살아있는 경우가 많다**(그림만 갈아엎고 소리는 유지). ID 경로나
> 사운드가 구버전이라고 해서 아트도 구버전일 거라 넘겨짚지 말 것 — 반드시 썸네일로 눈으로 확인한다.
>
> **옛날 아트가 필요하면 이 인덱스로는 못 찾는다. 구버전 클라이언트에서 추출해 업로드하는 수밖에 없다**
> (→ [`../resources/README.md`](../resources/README.md)).
>
> 그래도 이 인덱스가 쓸모 있는 경우: **사운드**(구버전 그대로인 것 많음), 아이콘, 그리고
> "현대 아트라도 쓸만한 것" 찾기 + 어떤 스킬이 무슨 요소를 갖고 있는지 파악.

생성: `node msw-dev-tools/dump_skill_pack_index.cjs` (재실행하면 전체 갱신)

| 파일 | 내용 | 언제 쓰나 |
|---|---|---|
| `스킬팩_목록.tsv` | 팩 1개 = 1행 (팩ID, 한글명, 영문명, 요소수, rel_path 목록) | 스킬이 있는지/무슨 요소를 가졌는지 |
| `스킬요소_목록.tsv` | 요소 1개 = 1행 (팩ID, 한글명, rel_path, 타입, **RUID**) | RUID를 바로 뽑을 때 |

현재 규모: **팩 5,299개 / 요소 58,265개** (sprite 39,118 · animationclip 14,883 · effect(사운드) 4,169 · voice 95)

## 핵심: 팩 ID = 메이플 원본 WZ 경로

```
skill/<직업img>.img/skill/<스킬ID>
```

`searchResources`에 한글 이름을 넣으면 **현대 리메이크판이 먼저 나오고 클래식 팩은 묻힌다.**
클래식 팩은 이름도 엉뚱하게 섞여 붙어 있다 (예: `1001005` = `용독술 / 슬래시 블러스트`).
그래서 이름 검색 대신 **WZ 스킬 ID로 조회**하는 게 정확하다.

```bash
# 스킬 ID를 알 때 (제일 확실)
grep "1001005" 스킬팩_목록.tsv

# 이름으로 훑을 때 (클래식/현대 전부 나옴)
grep "슬래시" 스킬팩_목록.tsv

# RUID까지 바로
grep "^skill/100.img/skill/1001005" 스킬요소_목록.tsv
```

|---|---|---|---|---|
| `100` 검사 | `200` 매지션 | `300` 궁수 | `400` 도적 | `500` 해적 |

| `111` 크루세이더 | `211` 메이지(불독) | `311` 레인저 | `411` 허미트 | `511` 버커니어 |
| `112` 히어로 | `212` 아크메이지(불독) | `312` 보우마스터 | `412` 나이트로드 | `512` 바이퍼 |

## ⚠ 커버리지는 스킬 단위로 들쭉날쭉하다

(위 경고대로 **아트는 전부 현대판**이다. 아래는 "그 스킬 항목 자체가 있느냐"의 이야기.)

직업 img는 42개 전부 있어도, **현대에서 삭제·재설계된 스킬은 원래 자리에 없다.**

실측 예 — `skill/100.img`(검사)에 있는 건 6개뿐이고 **파워 스트라이크(1001004)가 없다**:

```
1000003 아이언 바디 / 1000007 가디언 아머 / 1000009 워리어 마스터리
1001005 슬래시 블러스트 / 1001008 워리어 리프 / 1001010 리프 어택
```

### 원래 자리에 없을 때 → `800xxx.img`를 이름으로 뒤진다

`800xxx.img` 계열(**1,636개**, 최대 덩어리)에 기타/이벤트/구 스킬 항목이 모여 있다.
**단, 아트는 여기도 현대판이다**(80003316 파워 스트라이크 = 노란 반짝임 + 붉은 갈매기, 실측).

| 스킬 | 원래 자리 | 항목이 있는 곳 |
|---|---|---|
| 파워 스트라이크 | `100.img/1001004` ❌ 없음 | `skill/800033.img/skill/80003316` |
| 매직 클로 | `200.img/2001005` ❌ 없음 | `skill/800032.img/skill/80003294`, `800016.img/80001661` |
| 슬래시 블러스트 | `100.img/1001005` ✅ 있음 | 사용음 `afbb81b5…`(`_audio/Use`), 타격음 `ca35cb7d…`(`_audio/Hit`) |

이 프로젝트가 `800033.img/80003316`에서 실제로 쓰는 건 **`afterimage/*` 잔상뿐**이다(무기 스윙 궤적).
잔상 데이터는 스킬 아트와 별개로 버전 변화가 적어서 그대로 쓸 수 있었다. 정작 파워스트라이크의
**구버전 스킬 이펙트 자체는 외부에서 구해와야 했다**(`resources/파워스트라이크궤적/`).

```bash
# 아카이브에서 이름으로 찾기
grep "^skill/800" 스킬팩_목록.tsv | grep "파워"
```

### 그래도 없으면 → 외부에서 구해와 업로드

여기에도 없는 진짜 구버전 전용 리소스(구버전 클라이언트에만 있던 이펙트 등)는 직접 구해와서
`asset_create_account_resource_storage_item`로 업로드한다. 원본과 RUID 대응표는
[`../resources/README.md`](../resources/README.md) 규칙대로 `resources/<이름>/`에 보관한다.
(이 프로젝트의 `파워스트라이크궤적/`, `슬래시블러스트궤적/`가 그렇게 만든 것)

## rel_path 어휘 (요소가 뭔지 구분하는 키)

| rel_path | 내용 |
|---|---|
| `effect`, `effect0` | 시전 이펙트 본체 (animationclip) |
| `hit/0`, `hit/1`… | 타격 이펙트 |
| `icon`, `iconDisabled`, `iconMouseOver` | 스킬창 아이콘 3종 (거의 모든 팩에 있음) |
| `_audio/Use` | 사용 사운드 |
| `_audio/Hit` | 타격 사운드 |
| `_audio/Loop` | 지속 사운드 |
| `mob`, `summon/*` | 소환수 계열 |
| `special`, `keydown*` | 특수/차징 연출 |

```bash
# 예: 어떤 스킬의 사용 사운드 RUID만
grep "^skill/100.img/skill/1001005" 스킬요소_목록.tsv | grep "_audio/Use"
```

> RUID를 코드에 박기 전에 타입을 확인할 것. `SpriteRUID`에 animationclip을 넣으면 프레임이 전부
> 재생되고, sprite를 넣으면 그 한 장만 나온다. `effect` 타입은 **사운드**다(시각 이펙트 아님).
